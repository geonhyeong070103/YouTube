# 텍스트 그래픽 RPG (Pygame)

요청하신 방대한 기능 목록은 사실상 상용 MMORPG 한 편의 스펙에 가깝습니다.
그래서 이번 작업은 **"대략 만들어줘"** 라는 요청에 맞춰 아래 방식으로 진행했습니다.

- **실제로 동작하는 핵심 게임**: 캐릭터 생성 → 마을 → 필드 탐험 → 랜덤 인카운터/이벤트 →
  턴제 전투(속성/치명타/회피/상태이상/스킬) → 전리품/경험치 → 레벨업/스탯분배 →
  상점/인벤토리/장비 → 퀘스트 → 저장/불러오기 까지 **플레이 가능한 루프**가 완성되어 있습니다.
- **나머지 대형 시스템**(길드/PvP/멀티플레이/생활 콘텐츠 등)은 코드 골격 + 데이터 구조를 만들어
  두었고, 실제 UI 연결(main.py의 상태 하나 추가하는 수준)만 하면 이어붙일 수 있게 설계했습니다.
- 모드(mod) 폴더 구조를 만들어서, **동료와 기능을 나눠 개발할 때 서로의 코드를 건드리지 않고
  병렬로 작업**할 수 있게 했습니다 (아래 "협업 가이드" 참고).

## 실행 방법

```bash
pip install pygame
python main.py
```

- 방향키(↑/↓): 메뉴 이동
- Enter: 선택/확인
- 캐릭터 생성 화면: 이름 타이핑 → ←/→ 로 직업 선택 → Enter로 시작

## 폴더 구조

```
text_rpg/
├── main.py                 # 게임 루프 + 상태머신(화면 전환) + 렌더링
├── smoke_test.py            # 헤드리스 자동 플레이 테스트(핵심 플로우 회귀 검증용)
├── data/                    # 아이템/몬스터/퀘스트/월드맵 정의 (JSON) — 코드 수정 없이 콘텐츠 추가 가능
│   ├── items.json
│   ├── monsters.json
│   ├── quests.json
│   └── world.json
├── mods/                    # 모드(플러그인) 폴더 — game/mods.py 참고
│   └── example_guild_mod.py
├── saves/                   # 저장 파일 (자동 생성)
└── game/
    ├── constants.py         # 색상/화면 크기 등 전역 상수
    ├── enums.py             # 직업/속성/상태이상/희귀도/장비슬롯/게임상태 Enum
    ├── jobs.py               # 직업별 기초스탯/성장치/스킬트리/전직트리
    ├── character.py          # 플레이어: 레벨업/경험치/스탯분배/장비/상태이상
    ├── items.py               # 아이템/장비/소비/제작재료/퀘스트아이템 + 강화/랜덤옵션
    ├── inventory.py           # 인벤토리 정렬/자동정리/창고/장비비교
    ├── monsters.py            # 몬스터/도감/드롭테이블 (일반·엘리트·보스·월드보스)
    ├── combat.py              # 턴제 전투: 공격/스킬/아이템/도망, 치명타/회피, 속성상성, 상태이상
    ├── shop.py                # 구매/판매/시세변동/할인이벤트/비밀상인
    ├── quests.py              # 메인/서브/일일/반복 퀘스트, 진행도 갱신
    ├── world.py               # 마을/필드/던전(층별·랜덤·보스방·함정·보물방·비밀방)
    ├── npc.py                 # 상인/대장장이/연금술사/길드장/퀘스트NPC + 대화트리
    ├── events_random.py       # 습격/보물상자/행운/미스터리상인/숨겨진던전
    ├── weather_time.py        # 날씨/시간대/계절 + 전투 보정치
    ├── companions.py          # 동료(영입/육성/장비/호감도), 펫(포획/성장/진화)
    ├── reputation.py          # 도시/길드 평판, NPC 호감도
    ├── titles.py              # 칭호/업적 시스템
    ├── ethics.py              # 카르마(도둑질/현상금/감옥/비밀길드 접근)
    ├── endgame.py             # 무한던전/도전의탑/하드모드/시즌콘텐츠/랭킹
    ├── mods.py                # 모드 로더 (외부 py 파일을 안전하게 로드)
    ├── save_load.py           # 저장/불러오기/자동저장
    └── ui.py                  # 텍스트 렌더링 헬퍼(메뉴/상태창/전투로그/미니맵)
```

## 지금 완성되어 실제로 도는 기능

- 캐릭터 생성(이름/직업), 레벨/경험치/스탯 배분, 스킬 자동 습득
- 턴제 전투: 일반공격/스킬/아이템/도망, 치명타, 회피, 5종 상태이상, 6속성 상성
- 몬스터(일반/엘리트/보스/월드보스 데이터 포함), 몬스터 도감, 드롭 테이블
- 장비(무기/방어구/장신구) 장착/해제, 희귀도 6단계, 강화(+수치), 랜덤옵션, 세트효과(2피스 보너스)
- 인벤토리 정렬/자동정리/창고, 장비 스탯 비교 함수
- 상점(구매/판매/시세변동/할인이벤트), 비밀 상인 클래스
- 퀘스트(메인/서브/일일/반복), 진행도 자동 갱신, 보상 지급
- 월드맵(마을/숲/동굴/사막/설원/화산/성/던전/숨겨진 유적), 던전 층별 진행 + 랜덤 방 구성
  (일반/함정/보물방/비밀방/보스방)
- 랜덤 이벤트(습격/보물상자/행운/미스터리 상인/숨겨진 던전 발견)
- 날씨(맑음/비/눈/폭풍) + 시간(낮/밤)/계절 + 전투 보정치 훅
- 칭호/업적, 평판(도시/길드/NPC 호감도), 카르마(도둑질/현상금/감옥) 스텁 연동 가능
- 엔드콘텐츠 클래스(무한던전/도전의탑/하드모드/시즌/랭킹) — 아직 화면(state)엔 미연결
- 저장/불러오기(JSON), 자동저장(AutoSaver)
- 모드 로더: `mods/*.py`를 게임 시작 시 자동 로드

## 아직 "구조만" 있고 UI 연결이 필요한 부분

아래는 클래스/데이터는 준비되어 있지만 `main.py`의 상태머신에는 아직 화면이 없습니다.
동료와 나눠서 `GameState`에 항목을 추가하고 `handle_key`/`draw`에 분기만 붙이면 됩니다.

- 동료/펫 파티 UI (`companions.py`)
- 채집(벌목/채광/낚시/약초), 제작(무기/방어구/포션), 요리/농사/캠핑 등 생활 콘텐츠
- NPC 대화 선택지 UI (`npc.py`의 DialogueNode는 준비됨)
- 무한던전/도전의 탑/하드모드/시즌/랭킹 화면 (`endgame.py`)
- 퍼즐(미로/레버/암호/숨겨진 스위치), 뉴게임+, 퍼머데스/난이도 선택
- 멀티플레이(채팅/파티/거래/PvP/길드/레이드)는 별도 서버 통신 계층이 필요 (아래 참고)

## 멀티플레이에 대한 솔직한 안내

PvP·길드·레이드·거래·채팅 같은 멀티플레이 기능은 **서버(상태 동기화, 인증, 치트 방지)** 가
필요한 완전히 다른 규모의 작업입니다. 이 프로젝트는 싱글플레이 구조이며, 멀티플레이를 붙이려면
`game/mods.py`의 훅 구조를 확장해서 소켓 통신 모듈을 별도 모드로 얹는 방식을 권장합니다.
동료와 우선 싱글 콘텐츠부터 다지고, 멀티플레이는 별도 스프린트로 분리하시길 추천드립니다.

## 협업 가이드 (동료와 함께 작업 시)

1. **데이터만 추가하는 작업**(신규 아이템/몬스터/퀘스트/지역)은 `data/*.json`만 수정하면 됩니다.
   코드 충돌이 거의 없어 병렬 작업에 가장 안전합니다.
2. **새 시스템 추가**(예: 길드, 낚시)는 `game/` 에 새 모듈 파일을 만들고,
   `main.py`에는 `GameState`값 하나 + `_handle_xxx`/`_draw_xxx` 함수 한 쌍만 추가하는 패턴을 따르면
   기존 코드와 충돌 없이 병합하기 쉽습니다.
3. **실험적/독립적 기능**은 `mods/` 폴더에 `register(game_context)` 함수가 있는 파일로 작성하면
   `game/`, `main.py`를 아예 건드리지 않고 붙였다 뗄 수 있습니다.
4. VSCode Live Share 등으로 공유 작업 시, `data/*.json`과 `mods/*.py`부터 각자 맡아서
   충돌 범위를 줄이는 것을 추천합니다.

## 테스트

`smoke_test.py`는 pygame 창 없이(headless) 새 게임 → 필드 이동 → 전투 → 상점 → 저장/로드까지
자동으로 실행해 핵심 흐름이 깨지지 않았는지 확인하는 회귀 테스트입니다.

```bash
SDL_VIDEODRIVER=dummy python smoke_test.py
```
